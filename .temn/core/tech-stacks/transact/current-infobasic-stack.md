## Overview

This is the **current production technology stack** for T24 InfoBasic (jBC) development. All detailed patterns, conventions, and best practices have been migrated to the **InfoBasic Skill** to avoid duplication and provide a single source of truth.

**Use the InfoBasic Skill when:**

- Learning universal jBC patterns (control flow, loops, strings, multi-value fields)
- Understanding T24 File I/O patterns (F.READ, F.READU, F.WRITE, F.RELEASE)
- Following naming conventions (FN._, R._, ERR._, MY._, field equates)
- Avoiding anti-patterns (hard-coded positions, missing error handling, unreleased locks)
- Setting up $INSERT and $USING directives

**Use [CLAUDE.md](../../../../CLAUDE.md) when:**

- Working with AA Framework project structure
- Understanding repository organization and build process
- Setting up development workflow

**Use [CLAUDE_AA.md](./CLAUDE_AA.md) when:**

- Developing AA Framework components
- Working with COMMON blocks and framework accessors
- Implementing activity lifecycle processing

**Use [CLAUDE_TPH.md](./CLAUDE_TPH.md) when:**

- Developing TPH service layer components
- Using parameter naming conventions (iInput, oOutput, ioParameter)
- Implementing three-phase service structure
- Handling structured responses

## Core Technology Stack

### Banking Platform

- **Platform:** Temenos T24 Transact
- **Version:** (As deployed in production)
- **Core Banking:** T24 Core Banking System
- **Database:** jBASE (native T24 database system)
- **Runtime:** T24 Application Server

### Programming Language

- **Language:** jBC (jBase BASIC)
- **Paradigm:** Procedural programming
- **Compiler:** T24 jBC compiler
- **Optimization:** jcompile.optionk = true (enabled)

### Build System

- **Build Tool:** Apache Ant
- **Version:** 1.x+
- **Build Files:**
  - `build.xml` - Main build configuration
  - `build.properties` - Component-specific properties

**Build Targets:**

```bash
ant compile    # Compile jBC source
ant build      # Full build
ant release    # Create distribution
ant clean      # Clean build artifacts
```

### Dependency Management

- **Tool:** Apache Ivy
- **Version:** 2.x
- **Configuration:** `version.xml`
- **Dependencies:**
  - T24CommonDependencies (TEMENOS module)
  - T24BuildDependencies (build scripts)

### Component Structure

- **Format:** T24 Component
- **Metadata:** `.component` file (metamodel 2.0)
- **Organization:** Component-based architecture

## InfoBasic Skill Reference

All **universal jBC patterns** are documented in the **InfoBasic Skill**:

### Topics Covered in InfoBasic Skill:

- **Naming Conventions** - File names, record reads, error flags, local variables, constants, DAS variables
- **T24 Source File Naming** - Source files, include files, conversion routines
- **$USING vs $INSERT Directives** - When to use each, patterns and examples
- **Control Flow Patterns** - IF/THEN, BEGIN CASE, GOSUB, early returns
- **Loop Patterns** - LOOP/WHILE/REPEAT, FOR/NEXT, REMOVE
- **String Operations** - Concatenation, TRIM, CONVERT, FIELD
- **Multi-Value Field Patterns** - DCOUNT, LOCATE, field access
- **T24 File I/O Pattern** - F.READ, F.READU, F.WRITE, F.RELEASE with error handling
- **Anti-Patterns to Avoid** - Hard-coded positions, missing error handling, unreleased locks, raw jBC
- **Comment Standards** - File headers, section comments, modification history, inline comments

## Testing Standards

### Unit Tests

- **Format:** `.tut` files (T24 Unit Test format)
- **Naming:** `[ROUTINE.NAME]_[scenario].tut`
- **Location:** `Test/UnitTests/`
- **Framework:** T24 Unit Test Framework
- **Coverage Target:** >60% (aim for higher on critical routines)

### Integration Tests

- **Method:** T24 COB (Close of Business) simulation
- **Scope:** End-to-end scenarios
